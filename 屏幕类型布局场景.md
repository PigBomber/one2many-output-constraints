# 屏幕类型布局场景 输出约束

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-05-14
> 基于文档：屏幕类型布局场景（https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-screen-layout）
> 维度说明：本文件覆盖超大屏横屏、大屏横屏、大屏竖屏、大方形屏、直板机竖屏、直板机横屏、小方形屏、圆形屏八大屏幕类型的布局约束

## 约束列表

```yaml
constraints:

# =====================================================================
# 超大屏横屏（xl + sm）：PC/2in1
# =====================================================================

- id: SCR-MUST-01
  name: 超大屏横屏导航栏必须使用 SideBarContainer 侧边分级导航
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 xl 时，检查主导航组件是否使用 SideBarContainer 实现侧边分级导航栏（showSideBar 为 true），而非保留底部 Tabs 导航。xl 断点仍使用底部 Tabs 导致横向空间浪费即判定失败"

- id: SCR-MUST-02
  name: 超大屏横屏网格布局列数必须为 12 列或以上
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 xl 时，检查页面中 Grid 组件的 columnsTemplate 列数必须不少于 12（或使用 WaterFlow/List 的多列展示）。xl 断点下列数与 sm/md 断点相同即判定失败"

- id: SCR-MUST-03
  name: 超大屏横屏列表必须使用多列 lanes
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 xl 时，检查 List 组件的 lanes 值必须 >= 3（三列或以上）。xl 断点仍使用单列 lanes(1) 导致空间浪费即判定失败"

- id: SCR-MUST-04
  name: 超大屏横屏三层内容必须使用三分栏布局
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 xl 且页面包含三级层级内容时，检查是否组合使用 SideBarContainer（showSideBar:true）+ Navigation（mode:Split）实现三分栏。侧边栏 + 内容区双栏未达到三栏效果即判定失败"

# =====================================================================
# 大屏横屏（lg + sm）：Pad 横屏、三折叠三屏态
# =====================================================================

- id: SCR-MUST-05
  name: 大屏横屏导航栏必须切换为侧边 Tabs
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查主导航 Tabs 组件的 vertical 属性必须为 true、barPosition 必须为 BarPosition.Start。lg 断点仍保持 vertical:false 底部导航即判定失败"

- id: SCR-MUST-06
  name: 大屏横屏瀑布流必须使用多列 columnsTemplate
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查 WaterFlow 组件的 columnsTemplate 必须为 repeat(4, 1fr) 或更多列。lg 断点瀑布流仍为 2 列（与 sm 相同）即判定失败"

- id: SCR-MUST-07
  name: 大屏横屏轮播图必须增加 displayCount
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查 Swiper 组件的 displayCount 必须 >= 3。lg 断点仍 displayCount(1) 单张显示即判定失败"

- id: SCR-MUST-08
  name: 大屏横屏网格列数必须递增至 4 列或以上
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查 Grid 组件的 columnsTemplate 列数必须 >= 4。lg 断点 Grid 仍为 2 列（与 sm 相同）即判定失败"

- id: SCR-MUST-09
  name: 大屏横屏插图和文字必须使用左右布局
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查图文组合区域必须通过 GridRow/GridCol 实现左右分布（span 分配比例如 4:8 或 5:7），而非保持 sm 断点的上下堆叠。lg 断点图文仍为上下排列即判定失败"

- id: SCR-SHOULD-01
  name: 大屏横屏应展示侧边栏辅助内容
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 lg 时，检查是否在适当场景使用 SideBarContainer 展示辅助信息（showSideBar:true），充分利用横向空间。lg 断点无侧边栏展示辅助内容即判定为不满足"

# =====================================================================
# 大屏竖屏（md + lg）：Pad 竖屏
# =====================================================================

- id: SCR-MUST-10
  name: 大屏竖屏导航栏必须保持底部位置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 lg 时，检查 Tabs 组件的 barPosition 必须为 End、vertical 必须为 false。md+lg 场景使用侧边导航（vertical:true）不符合竖屏交互习惯即判定失败"

- id: SCR-MUST-11
  name: 大屏竖屏轮播图必须展示多个元素
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 时，检查 Swiper 组件的 displayCount 必须 >= 2。md 断点仍 displayCount(1) 单张显示未充分利用宽度即判定失败"

- id: SCR-MUST-12
  name: 大屏竖屏列表必须使用多列 lanes
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 时，检查 List 组件的 lanes 值必须 >= 2。md 断点仍使用 lanes(1) 单列未充分利用宽度即判定失败"

- id: SCR-SHOULD-02
  name: 大屏竖屏网格应使用多行多列
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 时，检查 Grid 组件的列数应 >= 3，充分利用竖屏的宽度空间。md 断点 Grid 仍为 2 列即判定为不满足"

# =====================================================================
# 大方形屏（md + md）：双折叠展开态、三折叠双屏态
# =====================================================================

- id: SCR-MUST-13
  name: 大方形屏导航栏必须保持底部位置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 md 时，检查 Tabs 组件 barPosition 为 End、vertical 为 false。md+md 方形屏使用侧边导航即判定失败"

- id: SCR-MUST-14
  name: 大方形屏 Navigation 必须支持双栏模式
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 md 时，检查 Navigation 组件的 mode 必须支持 NavigationMode.Split 双栏展示。md+md 仍使用 Stack 单栏即判定失败"

- id: SCR-SHOULD-03
  name: 大方形屏应使用双列列表或多列网格
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 md 时，检查列表 List 的 lanes 应 >= 2 或 Grid 的列数应 >= 3。md+md 仍使用单列列表即判定为不满足"

- id: SCR-SHOULD-04
  name: 大方形屏插图和文字应使用左右布局
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 md 时，检查图文组合区域应通过 GridRow/GridCol 实现左右分布。md+md 仍为上下堆叠即判定为不满足"

# =====================================================================
# 直板机竖屏（sm + lg）
# =====================================================================

- id: SCR-MUST-15
  name: 直板机竖屏导航栏必须使用底部 Tabs
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 时，检查主导航 Tabs 组件的 barPosition 必须为 End、vertical 必须为 false。sm 断点使用侧边导航即判定失败"

- id: SCR-MUST-16
  name: 直板机竖屏 Navigation 必须使用 Stack 单栏模式
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 时，检查 Navigation 组件的 mode 必须为 NavigationMode.Stack。sm 断点使用 Split 双栏会导致两侧内容被压缩即判定失败"

- id: SCR-SHOULD-05
  name: 直板机竖屏图文应使用上下布局
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 时，检查图文组合区域应使用上下堆叠布局（GridCol span 在 sm 断点占满全部列数如 4/4）。sm 断点强制左右布局导致内容过窄即判定为不满足"

# =====================================================================
# 直板机横屏（md + sm）
# =====================================================================

- id: SCR-MUST-17
  name: 直板机横屏必须支持双栏布局
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 sm 时，检查 Navigation 的 mode 必须为 NavigationMode.Split 或 SideBarContainer 的 showSideBar 为 true。md+sm 仍使用 Stack 单栏未充分利用横向空间即判定失败"

- id: SCR-MUST-18
  name: 直板机横屏插图和文字必须使用左右布局
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 sm 时，检查图文组合区域必须通过 GridRow/GridCol 或 Row 实现左右分布。md+sm 仍为上下堆叠即判定失败"

- id: SCR-SHOULD-06
  name: 直板机横屏应使用多列网格
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 md 且纵向断点为 sm 时，检查 Grid 组件的列数应 >= 3。md+sm 仍为 2 列即判定为不满足"

# =====================================================================
# 小方形屏（sm + md）：Pura X 外屏等
# =====================================================================

- id: SCR-MUST-19
  name: 小方形屏页面必须使用 Scroll 包裹保证内容可滑动
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 且纵向断点为 md 时，检查页面根容器是否使用 Scroll 组件包裹。小方形屏内容高度减半，无 Scroll 会导致内容截断不可见即判定失败"

- id: SCR-MUST-20
  name: 小方形屏自定义弹窗必须使用 constraintSize 限制最大高度
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 且纵向断点为 md 时，检查自定义弹窗的内容区是否使用 Scroll 包裹并设置 constraintSize({ maxHeight: '90%' })。弹窗内容未限制最大高度导致超出小方形屏范围即判定失败"

- id: SCR-SHOULD-07
  name: 小方形屏短视频页面侧边控件应支持滑动
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 且纵向断点为 md 时，检查短视频类页面的侧边控件区域是否使用 Scroll 组件包裹，并配合 Blank + displayPriority 控制上下留白的显隐。侧边控件区无 Scroll 导致部分控件被截断即判定为不满足"

- id: SCR-SHOULD-08
  name: 小方形屏应支持沉浸式浏览
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "当横向断点为 sm 且纵向断点为 md 时，检查内容列表页面是否实现了上滑隐藏标题栏/页签栏、下滑恢复的沉浸式浏览效果（通过监听滚动偏移量动态调整高度和透明度）。未实现沉浸式浏览即判定为不满足"

# =====================================================================
# 圆形屏（xs + sm）：智能手表
# =====================================================================

- id: SCR-MUST-21
  name: 圆形屏页面根容器必须设置 borderRadius 为 50%
  priority: P0
  rules:
  - target: '**/wearable/**/*.ets'
    llm: "当适配目标含 wearable 时，检查手表页面的最外层容器是否设置了 .borderRadius('50%')，确保内容适配圆形表盘。缺失 borderRadius 或值不为 50% 即判定失败"

- id: SCR-MUST-22
  name: 圆形屏列表必须使用 ArcList 而非 List
  priority: P0
  rules:
  - target: '**/wearable/**/*.ets'
    llm: "当适配目标含 wearable 时，检查手表模块中所有列表组件必须使用 ArcList + ArcListItem，不得使用普通 List/ListItem。使用普通 List 导致列表项两端被圆形屏幕裁切即判定失败"

- id: SCR-MUST-23
  name: 圆形屏页面切换必须使用 ArcSwiper
  priority: P0
  rules:
  - target: '**/wearable/**/*.ets'
    llm: "当适配目标含 wearable 时，检查手表页面间滑动切换场景是否使用 ArcSwiper 组件而非普通 Swiper。使用普通 Swiper 在圆形屏上无法正确显示轮播效果即判定失败"

- id: SCR-MUST-24
  name: 圆形屏内容布局必须避免边缘信息丢失
  priority: P0
  rules:
  - target: '**/wearable/**/*.ets'
    llm: "当适配目标含 wearable 时，检查 ArcListItem 内的内容根节点宽度设置，内容宽度应控制在约 90%（而非 100%），确保文字和图标不被圆形边缘裁切。内容宽度为 100% 导致两端被裁切即判定失败"

- id: SCR-SHOULD-09
  name: 圆形屏横向切屏应使用 ArcSwiper 分页展示
  priority: P1
  rules:
  - target: '**/wearable/**/*.ets'
    llm: "当适配目标含 wearable 时，检查内容量超过单屏的页面是否使用 ArcSwiper 实现横向切屏（左右滑动切换不同信息页），而非将所有内容垂直堆叠。垂直堆叠过多内容导致可读性差即判定为不满足"
```
