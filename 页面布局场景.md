# 页面布局场景 输出约束

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-05-14
> 基于文档：页面布局场景（https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-page-layout）
> 维度说明：本文件覆盖重复布局、分栏布局、挪移布局、缩进布局四大响应式布局方式的页面级约束

## 约束列表

```yaml
constraints:

# =====================================================================
# 重复布局 — 列表布局
# =====================================================================

- id: PGL-MUST-01
  name: 重复列表必须使用lanes属性
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查重复列表页面中 List 组件是否有 lanes 属性，且列间距（lanes 第二参数）设置为合理值（如 12vp）。"

- id: PGL-SHOULD-01
  name: 重复列表在 lg 断点应该使用三列
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查重复列表页面中 List 组件在 lg 断点下 lanes 值应该 >= 3（三列或以上），且列间距（lanes 第二参数）设置为合理值（如 12vp）。lg 断点仍为 lanes(1) 单列或 lanes(2) 双列即判定为不满足"

- id: PGL-SHOULD-02
  name: 重复列表 space 应该随断点递增
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查重复列表的 List 组件 space 属性，应该随断点递增（如 sm:8, md:12, lg:16）。使用 WidthBreakpointType(8, 12, 16, 16).getValue(widthBp) 为推荐写法。space 值不随断点变化即判定为不满足"

# =====================================================================
# 重复布局 — 瀑布流布局
# =====================================================================

- id: PGL-MUST-02
  name: 瀑布流在 sm 断点必须为双列、lg 断点必须为四列
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查瀑布流页面中 WaterFlow 组件的 columnsTemplate，sm 断点必须为 repeat(2, 1fr)，lg 断点必须为 repeat(4, 1fr)。sm 断点单列或 lg 断点仍为双列即判定失败"

# =====================================================================
# 重复布局 — 轮播布局
# =====================================================================

- id: PGL-MUST-03
  name: 轮播布局 displayCount 必须按 sm:1 / md:2 / lg:3 设置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查轮播页面中 Swiper 组件的 displayCount，sm 断点必须为 1，md 断点必须为 2，lg 断点必须为 3。使用 WidthBreakpointType(1, 2, 3, 3).getValue(widthBp) 为推荐写法。displayCount 不按断点递增即判定失败"

- id: PGL-MUST-04
  name: 轮播布局 sm 断点必须显示圆点指示器、md/lg 断点必须隐藏
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 Swiper 组件的 indicator 属性：sm 断点必须为 Indicator.dot()（显示圆点），md/lg 断点必须为 false（隐藏圆点）。多元素展示时保留圆点指示器或单元素时隐藏圆点即判定失败"

- id: PGL-SHOULD-03
  name: 轮播布局 md/lg 断点应设置前后边距露出前后项
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 Swiper 组件的 prevMargin 和 nextMargin：md 断点应设置 12vp，lg 断点应设置 64vp。md/lg 断点前后边距均为 0 未提供滑动暗示即判定为不满足"

# =====================================================================
# 重复布局 — 网格布局
# =====================================================================

- id: PGL-MUST-05
  name: 网格布局列数必须按 sm:2 / md:3 / lg:4 递增
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查网格页面中 Grid 组件的 columnsTemplate，sm 断点必须为 repeat(2, 1fr)，md 断点必须为 repeat(3, 1fr)，lg 断点必须为 repeat(4, 1fr)。使用 `repeat(${new WidthBreakpointType(2,3,4,4).getValue(widthBp)}, 1fr)` 为推荐写法。列数不随断点递增即判定失败"

- id: PGL-SHOULD-04
  name: 网格布局应设置 columnsGap 和 rowsGap
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 Grid 组件是否设置了 columnsGap(12) 和 rowsGap(12)。网格布局缺失 gap 设置导致子项紧密排列无间距即判定为不满足"

# =====================================================================
# 分栏布局 — 侧边栏
# =====================================================================

- id: PGL-MUST-06
  name: 侧边栏 sm 断点必须隐藏且使用 Overlay 模式
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查侧边栏页面的 SideBarContainer 组件：sm 断点 showSideBar 必须为 false，且 SideBarContainerType 必须为 Overlay（浮层模式，侧边栏浮在内容区上）。sm 断点 showSideBar 为 true 或使用 Embed 模式即判定失败"

- id: PGL-MUST-07
  name: 侧边栏 md/lg 断点必须显示且使用 Embed 模式
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查侧边栏页面的 SideBarContainer 组件：md/lg 断点 showSideBar 必须为 true，且 SideBarContainerType 必须为 Embed（嵌入模式，侧边栏和内容区并列展示）。md/lg 断点使用 Overlay 模式即判定失败"

- id: PGL-MUST-08
  name: 侧边栏宽度必须按断点差异化设置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 SideBarContainer 的 sideBarWidth 属性必须按断点设置不同值：sm 为 '80%'（浮层时占大部分宽度），md 为 '50%'，lg 为 '40%'。使用 WidthBreakpointType('80%','50%','40%','40%').getValue(widthBp) 为推荐写法。所有断点 sideBarWidth 相同即判定失败"

# =====================================================================
# 分栏布局 — 聊天典型场景
# =====================================================================

- id: PGL-MUST-09
  name: 聊天场景双栏时内容扩展页必须切回 Stack 单栏
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查社交类应用的聊天页面：双栏模式下（md/lg 断点）点击右侧内容区的扩展链接（如商品链接）时，Navigation.mode 必须从 Split 切回 Stack 全屏展示扩展内容。双栏模式下扩展内容仍在分栏右侧展示而非全屏即判定失败"

- id: PGL-SHOULD-05
  name: 聊天场景返回时应恢复 Split 双栏模式
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查聊天页面从扩展内容返回时，Navigation.mode 是否恢复为 Split 双栏。返回后仍保持 Stack 单栏即判定为不满足"

# =====================================================================
# 分栏布局 — 三分栏
# =====================================================================

- id: PGL-MUST-10
  name: 三分栏 sm 断点必须为单栏无侧边栏
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查三分栏页面在 sm 断点下：SideBarContainer.showSideBar 必须为 false，SideBarContainerType 必须为 Overlay，Navigation.mode 必须为 Stack。sm 断点下任一条件不满足即判定失败"

- id: PGL-MUST-11
  name: 三分栏 md 断点必须为双栏无侧边栏
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查三分栏页面在 md 断点下：SideBarContainer.showSideBar 必须为 false，Navigation.mode 必须为 Split（双栏），navBarWidth 应为 '50%'。md 断点 showSideBar 为 true 或 Navigation.mode 为 Stack 即判定失败"

- id: PGL-MUST-12
  name: 三分栏 lg 断点必须为三栏并排
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查三分栏页面在 lg 断点下：SideBarContainer.showSideBar 必须为 true，SideBarContainerType 必须为 Embed，sideBarWidth 应为 '20%'，Navigation.mode 必须为 Split，navBarWidth 应为 '30%'。lg 断点未同时满足所有条件即判定失败"

# =====================================================================
# 分栏布局 — 邮箱典型场景
# =====================================================================

- id: PGL-SHOULD-06
  name: 邮箱场景 lg 断点应默认显示三栏
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查邮箱类应用在 lg 断点下是否默认显示三栏：左侧账户信息（SideBarContainer 侧边栏）、中间收件箱（Navigation 导航栏）、右侧邮件详情（Navigation 内容区）。lg 断点默认隐藏侧边栏需手动打开即判定为不满足"

# =====================================================================
# 分栏布局 — 日历典型场景
# =====================================================================

- id: PGL-SHOULD-07
  name: 日历场景单栏时应优先展示导航区而非内容区
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查日历日程类应用在 sm 单栏模式下，Navigation 是否优先展示日历（导航栏内容）而非日程详情（内容区）。实现方式：在 onNavigationModeChange 中检测到 Stack 模式时调用 pathStack.clear() 清空内容区路由。单栏时默认显示空白详情页而非日历即判定为不满足"

# =====================================================================
# 挪移布局 — 插图和文字组合
# =====================================================================

- id: PGL-MUST-13
  name: 图文组合 sm 断点必须上下排列、lg 断点必须左右排列
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查图文组合页面的 GridRow/GridCol 配置：sm 断点插图和文字 span 必须均为 4（占满全部 4 列，上下排列），lg 断点插图 span 为 4、文字 span 为 8（左右排列）。sm 断点即左右排列或 lg 断点仍上下排列即判定失败"

- id: PGL-MUST-14
  name: 图文组合 GridRow 的 columns 必须按 sm:4 / md:8 / lg:12 设置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查图文组合页面 GridRow 的 columns 配置必须为 { sm:4, md:8, lg:12 } 或等价写法。栅格列数不随断点递增即判定失败"

# =====================================================================
# 挪移布局 — 底部/侧边导航
# =====================================================================

- id: PGL-MUST-15
  name: sm/md 断点导航必须为底部一级导航 + 顶部二级页签
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查分级导航页面在 sm/md 断点下：Tabs 组件 barPosition 必须为 End、vertical 必须为 false（底部横向导航），二级页签通过顶部 TabContent 内嵌 Tabs 实现。sm/md 断点使用侧边导航即判定失败"

- id: PGL-MUST-16
  name: lg 断点导航必须为侧边一级导航 + 顶部二级页签
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查分级导航页面在 lg 断点下：Tabs 组件 barPosition 必须为 Start、vertical 必须为 true、barWidth 必须为 96vp、barHeight 必须为 '100%'。lg 断点仍使用底部导航即判定失败"

- id: PGL-MUST-17
  name: xl 断点或 PC 设备必须使用 SideBarContainer 展示分级导航
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查分级导航页面在 xl 断点或 2in1 设备上是否使用 SideBarContainer(SideBarContainerType.Embed) 展示一级和二级导航目录，替代 Tabs 组件。xl 断点仍使用 Tabs 无法展示两级导航结构即判定失败"

- id: PGL-SHOULD-08
  name: xl 断点 SideBarContainer 应设置固定宽度
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 xl 断点下 SideBarContainer 的 sideBarWidth、minSideBarWidth、maxSideBarWidth 是否设置为相同的固定值（如 240vp），确保侧边栏宽度不随窗口变化。xl 断点侧边栏使用百分比宽度导致内容不稳定即判定为不满足"

# =====================================================================
# 缩进布局 — 单列列表
# =====================================================================

- id: PGL-MUST-18
  name: 缩进布局 GridCol span 必须按 sm:4 / md:6 / lg:8 设置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查缩进布局的单列表页面的 GridCol 组件 span 值必须随断点递增：sm 断点占 4 列（占满），md 断点占 6 列，lg 断点占 8 列。所有断点 span 值相同（如均为 12）未实现居中缩进效果即判定失败"

- id: PGL-MUST-19
  name: 缩进布局 GridCol offset 必须随断点递增
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查缩进布局的单列表页面的 GridCol 组件 offset 值必须随断点递增：sm 断点 offset 为 0，md 断点 offset 为 1（两侧各留 1 列），lg 断点 offset 为 2（两侧各留 2 列）。offset 不随断点递增或始终为 0 即判定失败"

- id: PGL-MUST-20
  name: 缩进布局 GridRow 的 columns 必须按 sm:4 / md:8 / lg:12 设置
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查缩进布局的单列表页面 GridRow 的 columns 配置必须为 { sm:4, md:8, lg:12 }，确保与 span 和 offset 配合实现正确的居中缩进。columns 值不符合推荐值会导致 span+offset 超出总列数即判定失败"
```
