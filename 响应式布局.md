# 响应式布局 输出约束

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-05-14
> 基于文档：响应式布局（https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-responsive-layout）
> 维度说明：本文件覆盖断点系统、媒体查询、栅格组件、纵向断点等响应式基础能力的约束

## 约束列表

```yaml
constraints:

# =====================================================================
# 断点定义与使用（Breakpoint Definition & Usage）
# =====================================================================

- id: RSP-MUST-01
  name: 横向断点划分范围必须符合系统推荐值
  priority: P0
  rules:
  - target: '**/*.ets'
    llm: "检查工程中自定义断点系统或 WidthBreakpointType 工具类的断点边界定义，横向断点划分必须为 xs:(0,320)、sm:[320,600)、md:[600,840)、lg:[840,1440)、xl:[1440,+∞)。若使用 GridRow 的 breakpoints.value，值必须为 ['320vp','600vp','840vp','1440vp']。断点边界值与系统推荐不一致即判定失败"

- id: RSP-MUST-02
  name: 布局条件分支必须使用断点枚举值而非硬编码宽度
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查页面代码中所有布局条件判断（if/else、三元表达式），必须使用 WidthBreakpoint.WIDTH_SM/MD/LG/XL 枚举或 BreakpointType 的 sm/md/lg/xl 字段，不得使用 if(width > 600) 或 if(vp > 840) 等硬编码数值比较。存在硬编码宽度值做布局判断即判定失败"

- id: RSP-MUST-03
  name: WidthBreakpointType 工具类必须覆盖 sm/md/lg/xl 四个断点
  priority: P0
  rules:
  - target: '**/WidthBreakpointType.ets'
    llm: "检查 WidthBreakpointType 工具类定义，构造函数必须接受 sm、md、lg、xl 四个参数，getValue 方法必须根据 WidthBreakpoint 枚举正确返回对应值。缺失任一断点参数或 getValue 分支不完整即判定失败"

- id: RSP-MUST-04
  name: 页面组件必须通过断点系统获取当前断点值
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查页面中需要根据断点切换布局的组件，是否通过 @ObjectLink mainWindowInfo.widthBp、BreakpointSystem 的 BreakpointState、或 @Env 系统断点变量获取断点值。自行计算窗口宽度除以密度来推导断点即判定失败"

# =====================================================================
# 纵向断点（Vertical Breakpoint）
# =====================================================================

- id: RSP-MUST-05
  name: 折叠屏和特殊窗口场景必须同时使用纵向断点
  priority: P0
  rules:
  - target: '**/*.ets'
    llm: "当适配目标含 foldable 或存在类方形窗口场景时，检查断点系统是否同时提供纵向断点（HeightBreakpoint: HEIGHT_SM/MD/LG，按窗口高宽比划分）。仅使用横向断点无法区分 Pura X 外屏（sm+md）和直板机竖屏（sm+lg）即判定失败"

- id: RSP-MUST-06
  name: 横向窗口判断必须使用纵向断点为 sm 的条件
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查需要区分横向窗口的逻辑，判断条件必须使用 currentHeightBreakpoint === HeightBreakpoint.HEIGHT_SM（高宽比<0.8），而非 deviceType 或屏幕方向 API。类方形窗口判断必须使用 HEIGHT_MD（高宽比在 [0.8,1.2)）。使用非断点方式判断窗口形态即判定失败"

- id: RSP-MUST-07
  name: 类方屏旋转方案必须通过纵向断点 md 触发
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查类方屏（如 Mate X5 展开态）窗口旋转逻辑，设置 window.setPreferredOrientation(AUTO_ROTATION_RESTRICTED) 的条件必须包含纵向断点为 md 的判断（HeightBreakpoint.HEIGHT_MD），而非使用 isFoldable() + getFoldStatus() 组合。使用设备类型或折叠状态判断旋转策略即判定失败"

# =====================================================================
# 断点系统注册与生命周期
# =====================================================================

- id: RSP-MUST-08
  name: 窗口尺寸监听必须基于 on('windowSizeChange')
  priority: P0
  rules:
  - target: '**/*.ets'
    llm: "检查断点更新逻辑是否注册在 window.on('windowSizeChange') 回调中，并在回调内通过 uiContext.getWindowWidthBreakpoint() 和 uiContext.getWindowHeightBreakpoint() 重新获取断点值。使用 foldStatusChange 或设备方向 API 驱动断点更新即判定失败"

- id: RSP-MUST-09
  name: WindowUtil 必须在 loadContent 回调后调用 updateWindowInfo
  priority: P0
  rules:
  - target: '**/EntryAbility.ets'
    llm: "检查 EntryAbility 的 onWindowStageCreate 方法，WindowUtil 初始化（setUIContext + updateWindowInfo）必须在 windowStage.loadContent 的回调内执行（确保 UIContext 已创建）。在 loadContent 之前调用 updateWindowInfo 会导致断点监听注册失败即判定失败"

- id: RSP-MUST-10
  name: WindowUtil 对象必须持久化存储到 AppStorage
  priority: P0
  rules:
  - target: '**/EntryAbility.ets'
    llm: "检查 EntryAbility 是否在 onWindowStageCreate 中将 WindowUtil 实例通过 AppStorage.setOrCreate('windowUtil', windowUtil) 持久化。页面组件必须通过 @StorageLink('windowUtil') 获取。WindowUtil 仅作为局部变量传递导致子组件无法获取断点信息即判定失败"

# =====================================================================
# 栅格组件（GridRow / GridCol）
# =====================================================================

- id: RSP-MUST-11
  name: GridRow 的 breakpoints.value 必须与系统断点一致
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查所有 GridRow 组件的 breakpoints.value 配置，值必须为 ['320vp','600vp','840vp','1440vp']。自定义栅格断点阈值（如 ['400vp','700vp']）会导致与全局断点系统不同步即判定失败"

- id: RSP-SHOULD-3
  name: GridCol 的 span 必须按断点声明不同值
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 GridCol 组件的 span 配置，对于需要响应式适配的栅格子组件，sm/md/lg 断点下 span 值必须不同（如 sm:4, md:4, lg:8 而非所有断点均为 12）。所有断点 span 值相同的 GridCol 不构成响应式布局即判定失败"

- id: RSP-SHOULD-4
  name: GridRow columns 必须按断点递增
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 GridRow 组件的 columns 配置，列数必须随断点从小到大递增（如 sm:4, md:8, lg:12）。较大断点的列数小于较小断点或所有断点列数相同（不构成响应式）即判定失败"

- id: RSP-SHOULD-01
  name: 栅格组件应设置合理的 gutter 值
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查 GridRow 组件是否配置了 gutter 属性（x 和 y 方向），建议 sm:8/12、md:16、lg:24。缺失 gutter 会导致栅格子元素之间无间距即判定为不满足"

- id: RSP-SHOULD-02
  name: GridCol offset 应用于实现缩进布局
  priority: P1
  rules:
  - target: '**/pages/*.ets'
    llm: "检查需要在大屏居中显示的单列内容区域（如设置页、详情表单），是否通过 GridCol 的 offset 属性实现两侧留白（如 md:offset=1, lg:offset=2），而非手动设置左右 padding/margin 模拟。使用硬编码 margin 而非栅格 offset 实现缩进即判定为不满足"

# =====================================================================
# 媒体查询（MediaQuery）
# =====================================================================

- id: RSP-MUST-12
  name: BreakpointSystem 单例必须通过 getInstance 获取
  priority: P0
  rules:
  - target: '**/BreakpointSystem.ets'
    llm: "检查 BreakpointSystem 类是否实现了单例模式（private constructor + static getInstance），外部必须通过 BreakpointSystem.getInstance() 获取实例。允许 new BreakpointSystem() 创建多实例会导致监听器重复注册即判定失败"

- id: RSP-MUST-13
  name: BreakpointSystem 必须在 aboutToAppear 中 attach 并 start
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查使用 BreakpointSystem 的页面组件，aboutToAppear 中必须调用 BreakpointSystem.getInstance().attach(state) 注册 BreakpointState 并调用 start()。仅在 aboutToAppear 中 attach 但未调用 start 即判定失败"

- id: RSP-MUST-14
  name: BreakpointSystem 必须在 aboutToDisappear 中 detach 并 stop
  priority: P0
  rules:
  - target: '**/pages/*.ets'
    llm: "检查使用 BreakpointSystem 的页面组件，aboutToDisappear 中必须调用 detach 移除所有 BreakpointState 并调用 stop 清理监听器。缺失清理逻辑导致媒体查询监听器泄漏即判定失败"
```
