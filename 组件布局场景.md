# 组件布局场景 输出约束

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-05-14
> 基于文档：组件布局场景（https://developer.huawei.com/consumer/cn/doc/best-practices/bpta-multi-device-component-layout）
> 维度说明：本文件覆盖 List、WaterFlow、Swiper、Grid、SideBarContainer、Navigation、Tabs、GridRow/GridCol、Flex、Row/Column、Blank、Scroll 等响应式组件及通用自适应属性（layoutWeight、displayPriority、aspectRatio）在不同断点下的属性配置约束

## 约束列表

```yaml
constraints:
  # =====================================================================
  # List 组件 — 重复列表布局
  # =====================================================================
  - id: CMP-MUST-01
    name: List lanes 必须按断点递增设置列数
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查使用 List 组件展示重复内容的页面，lanes 属性必须根据断点设置对应list组件布局的列数或行数，及列间距，并且断点应该根据断点值成递增趋势。所有断点 lanes 值相同或 sm 断点即使用多列即判定失败"

  - id: CMP-SHOULD-01
    name: List space 必须按断点设置不同间距
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 List 组件的 space 属性是否根据断点设置不同值。所有断点 space 值相同即判定失败"

  - id: CMP-SHOULD-02
    name: List 多列时 divider 应设为 undefined
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 List 组件在 lanes >= 2 时是否将 .divider() 设为 undefined。多列 List 保留 divider 会导致分割线在列间错乱即判定为不满足"

  # =====================================================================
  # WaterFlow 组件 — 重复瀑布流布局
  # =====================================================================

  - id: CMP-MUST-02
    name: WaterFlow columnsTemplate 必须按断点递增列数
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 WaterFlow 组件的 columnsTemplate 是否根据断点设置递增列数（如 sm:repeat(2,1fr), md:repeat(3,1fr), lg:repeat(4,1fr)）。所有断点列数相同即判定失败"

  - id: CMP-SHOULD-03
    name: WaterFlow 应设置 columnsGap 和 rowsGap
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 WaterFlow 组件是否设置了 columnsGap 和 rowsGap（建议值 12vp），确保瀑布流子项间有合理间距。缺失 gap 属性导致子项紧密排列即判定为不满足"

  - id: CMP-SHOULD-04
    name: 在适应屏幕宽度变化的场景下，需要动态切换列数
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "若要动态设置列数，建议采用瀑布流的移动窗口布局模式，即取值为WaterFlowLayoutMode枚举说明中的SLIDING_WINDOW，从而实现更快速的列数转换"

  - id: CMP-SHOULD-05
    name: 通过itemConstraintSize设置不同断点下WaterFlow所有子组件的约束尺寸
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "在图片、卡片瀑布流上，设置itemConstraintSize属性用来保证内容可读性，防止卡片过大导致瀑布流布局异常。itemConstraintSize属性建议根据断点值动态调整约束"
  # =====================================================================
  # Swiper 组件 — 重复轮播视图
  # =====================================================================

  - id: CMP-MUST-03
    name: Swiper displayCount 必须按断点递增
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Swiper 组件的 displayCount 是否根据断点设置递增值（如 sm:1, md:2, lg:3）。所有断点 displayCount 相同即判定失败"

  - id: CMP-MUST-04
    name: Swiper 在多元素展示时必须隐藏圆点指示器
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Swiper 组件在 displayCount >= 2 时 indicator 必须设为 false。多元素展示时仍显示圆点指示器会导致指示点位置错乱即判定失败"

  - id: CMP-SHOULD-06
    name: Swiper 多元素展示时应设置前后边距
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Swiper 组件在 displayCount >= 2 时是否设置了 prevMargin 和 nextMargin（如 md:12vp, lg:64vp），露出前后项的一部分以提示可滑动。多元素展示时 prevMargin 和 nextMargin 均为 0 即判定为不满足"

  # =====================================================================
  # Grid 组件 — 重复网格
  # =====================================================================

  - id: CMP-MUST-05
    name: Grid columnsTemplate 必须按断点递增列数
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Grid 组件的 columnsTemplate 是否根据断点设置递增列数（如 sm:repeat(2,1fr), md:repeat(3,1fr), lg:repeat(4,1fr)）。所有断点列数相同即判定失败"

  - id: CMP-SHOULD-07
    name: Grid 应设置 columnsGap 和 rowsGap
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Grid 组件是否设置了 columnsGap 和 rowsGap（建议值 12vp）。缺失 gap 属性导致网格子项紧密排列即判定为不满足"

  # =====================================================================
  # SideBarContainer 组件 — 侧边栏
  # =====================================================================

  - id: CMP-MUST-06
    name: SideBarContainer showSideBar 必须按断点动态控制
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 SideBarContainer 组件的 showSideBar 属性是否根据断点动态设置：sm 断点默认 false（隐藏侧边栏），md/lg 断点默认 true（显示侧边栏）。所有断点 showSideBar 值固定不变即判定失败"

  - id: CMP-MUST-07
    name: SideBarContainer sideBarWidth 必须按断点设置不同宽度
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 SideBarContainer 组件的 sideBarWidth 是否根据断点设置不同值（如 sm:'80%', md:'50%', lg:'40%'）。所有断点 sideBarWidth 值相同即判定失败"

  - id: CMP-MUST-08
    name: SideBarContainer 类型必须按断点区分 Overlay 和 Embed
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 SideBarContainer 组件的构造参数是否根据断点区分：sm 使用 SideBarContainerType.Overlay（浮层模式），md/lg 使用 SideBarContainerType.Embed（嵌入模式）。所有断点使用相同类型即判定失败"

  # =====================================================================
  # Navigation 组件 — 单/双栏
  # =====================================================================

  - id: CMP-MUST-09
    name: Navigation mode 必须按断点切换 Stack 和 Split
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Navigation 组件的 mode 属性是否根据断点动态设置：sm 断点使用 NavigationMode.Stack（单栏），md/lg 断点使用 NavigationMode.Split（双栏）。所有断点使用相同 mode 即判定失败"

  - id: CMP-SHOULD-08
    name: Navigation 双栏时 navBarWidth 应设置为合理比例
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Navigation 组件在 Split 模式下是否设置了 navBarWidth（建议 md:'50%', lg:'40%' 或 280vp-400vp 固定值）。双栏时未设置 navBarWidth 导致导航栏占比不合理即判定为不满足"

  # =====================================================================
  # SideBarContainer + Navigation — 三分栏
  # =====================================================================

  - id: CMP-MUST-10
    name: 三分栏组合必须同时控制 SideBarContainer 和 Navigation 属性
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查三分栏布局中 SideBarContainer 和 Navigation 组件是否同时根据断点设置：sm 断点 SideBarContainer.showSideBar 为 false + Navigation.mode 为 Stack；md 断点 SideBarContainer.showSideBar 为 false + Navigation.mode 为 Split；lg 断点 SideBarContainer.showSideBar 为 true + Navigation.mode 为 Split。仅控制其中一个组件的属性即判定失败"

  - id: CMP-MUST-11
    name: 三分栏 lg 断点 SideBarContainer 必须使用 Embed 模式
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查三分栏布局在 lg 断点下 SideBarContainer 的类型是否为 SideBarContainerType.Embed（嵌入模式），确保三栏并排显示。lg 断点仍使用 Overlay（浮层模式）导致侧边栏遮挡内容区即判定失败"

  # =====================================================================
  # Tabs 组件 — 底部/侧边导航
  # =====================================================================

  - id: CMP-MUST-12
    name: Tabs vertical 必须按断点切换横向和纵向
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查主导航 Tabs 组件的 vertical 属性是否根据断点动态设置：sm/md 断点 vertical 为 false（底部横向导航），lg 断点 vertical 为 true（侧边纵向导航）。所有断点 vertical 值相同即判定失败"

  - id: CMP-MUST-13
    name: Tabs barPosition 必须按断点切换 End 和 Start
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Tabs 组件的 barPosition 属性是否根据断点动态设置：sm/md 断点为 BarPosition.End（底部），lg 断点为 BarPosition.Start（左侧）。barPosition 与 vertical 不匹配（如 vertical:true 但 barPosition:End）即判定失败"

  - id: CMP-MUST-14
    name: Tabs barWidth 和 barHeight 必须按断点设置
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Tabs 组件的 barWidth 和 barHeight 是否根据断点动态设置，barWidth/barHeight 在所有断点保持固定值即判定失败"

  - id: CMP-MUST-15
    name: xl 断点或 PC 设备必须使用 SideBarContainer 替代 Tabs 实现分级导航
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "当横向断点为 xl 或设备为 2in1 时，检查是否使用 SideBarContainer 实现分级导航（包含一级和二级导航目录），而非仅使用 Tabs 组件。xl 断点仍仅使用 Tabs 无法展示分级导航结构即判定失败"

  # =====================================================================
  # GridRow / GridCol — 挪移布局与缩进布局
  # =====================================================================

  - id: CMP-MUST-16
    name: GridRow columns 必须按断点递增列数
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 GridRow 组件的 columns 属性是否按断点递增（如 { sm:4, md:8, lg:12, xl:12 }）。列数不随断点递增或所有断点列数相同即判定失败"

  - id: CMP-MUST-17
    name: GridCol span 必须按断点声明不同的占据列数
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查需要响应式适配的 GridCol 组件，sm/md/lg 断点下 span 值必须不同。如插图区域 sm:4(占满), md:4(半屏), lg:4(1/3)，文字区域 sm:4, md:4, lg:8。所有断点 span 相同无法实现挪移效果即判定失败"

  - id: CMP-SHOULD-09
    name: GridRow 应配置 gutter 属性
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 GridRow 组件是否配置了 gutter.x 和 gutter.y 属性。缺失 gutter 导致栅格子组件之间无间距即判定为不满足"

  - id: CMP-SHOULD-10
    name: 缩进布局应使用 GridCol offset 实现居中留白
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查需要在大屏居中展示的内容区域是否通过 GridCol 的 offset 属性实现两侧留白（如 md:offset=1, lg:offset=2）。使用硬编码 margin/padding 模拟缩进而非栅格 offset 即判定为不满足"

  # =====================================================================
  # Flex 组件 — 拉伸/均分/折行布局
  # =====================================================================

  - id: CMP-MUST-18
    name: Flex 拉伸布局必须合理设置 flexGrow 和 flexShrink
    priority: P0
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Flex 拉伸布局中，需要随容器伸缩的内容区子组件是否设置了 flexGrow(>=1)，需要固定尺寸的留白区子组件是否设置了 flexShrink(0)。所有子组件使用相同的 flexGrow/flexShrink 值或未设置拉伸/收缩属性即判定失败"

  - id: CMP-SHOULD-11
    name: Flex 均分布局应使用 FlexAlign.SpaceEvenly
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Flex 等间距均分布局（如工具栏、菜单栏）是否将 justifyContent 属性设置为 FlexAlign.SpaceEvenly。使用固定 margin/padding 手动计算间距来实现均分即判定为不满足"

  - id: CMP-SHOULD-12
    name: Flex 折行布局应设置 wrap 为 FlexWrap.Wrap
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查内容可能溢出容器宽度的 Flex 布局是否设置了 wrap: FlexWrap.Wrap。使用固定列数或手动计算换行位置即判定为不满足"

  # =====================================================================
  # Row/Column 组件 — 占比/隐藏能力
  # =====================================================================

  - id: CMP-SHOULD-13
    name: Row/Column 占比布局中子组件应使用 layoutWeight
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Row/Column/Flex 容器中需要按比例分配空间的兄弟子组件是否使用 layoutWeight 属性或百分比（%）形式的 width/height。使用固定 vp 值导致无法随容器自适应即判定为不满足"

  - id: CMP-SHOULD-14
    name: Row/Column 中随尺寸变化的显隐子组件应设置 displayPriority
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Row/Column/Flex 容器中需要随容器尺寸变化自动显示或隐藏的子组件是否设置了 displayPriority 属性控制显隐优先级。使用 if 条件判断配合断点手动控制显隐而非 displayPriority 即判定为不满足"

  # =====================================================================
  # Blank 组件 — 填充剩余空间
  # =====================================================================

  - id: CMP-SHOULD-15
    name: Row/Column/Flex 中固定元素间的空白应使用 Blank 组件填充
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查 Row/Column/Flex 中固定元素（如文字与开关）之间的空白区域是否使用 Blank 组件填充剩余空间。使用固定 width/height 的空容器模拟空白区域即判定为不满足"

  # =====================================================================
  # Scroll 组件 — 延伸能力
  # =====================================================================

  - id: CMP-SHOULD-16
    name: 横向延伸内容应使用 Scroll 配合 Row 实现
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查横向可延伸的内容列表是否使用 Scroll 组件配合 Row/Column（设置 scrollable(ScrollDirection.Horizontal)）或使用 List 的 listDirection(Axis.Horizontal) 实现延伸能力。内容超出容器宽度但未使用可滚动容器即判定为不满足"

  # =====================================================================
  # 容器子组件 — 缩放能力（aspectRatio）
  # =====================================================================

  - id: CMP-SHOULD-17
    name: 需要保持宽高比的容器子组件应设置 aspectRatio
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查需要随容器尺寸变化但保持固定宽高比的子组件（如图片容器、卡片）是否设置了 aspectRatio 属性，同时宽或高使用百分比或自适应尺寸。宽高都使用固定 vp 值且未设置 aspectRatio 即判定为不满足"

  - id: CMP-SHOULD-18
    name: 容器子组件的 aspectRatio 应根据断点设置不同宽高比
    priority: P1
    rules:
      - target: "**/pages/*.ets"
        llm: "检查设置了 aspectRatio 的子组件是否根据断点设置不同宽高比（如 sm 断点使用 4:3 适配竖屏，lg 断点使用 16:9 适配宽屏）。所有断点 aspectRatio 值相同即判定为不满足"
```
